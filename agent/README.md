# NOFS Agent — агент на ПК (Windows)

Трей-приложение (.NET 8, WinForms): WebSocket-сервер для планшета, UDP-автопоиск,
метрики через LibreHardwareMonitor, медиа-сессия Windows, контекст активного окна,
макросы, git и GitHub.

## Сборка и запуск

Нужен [.NET 8 SDK](https://dotnet.microsoft.com/download/dotnet/8.0). Затем:

```powershell
cd agent\NofsAgent
dotnet publish -c Release -r win-x64 --self-contained false
# exe: bin\Release\net8.0-windows10.0.19041.0\win-x64\publish\NofsAgent.exe
```

Или просто `dotnet run` из папки проекта для отладки.

**Права администратора не нужны.** Сервер поднят на обычном TCP-сокете
(`TcpListener`), а не на `HttpListener`/http.sys — порт открывается без
`netsh http add urlacl` и без элевации. Можно запускать агента как обычного
пользователя.

Единственное, что деградирует без админки — температуры CPU/GPU (нужны
LibreHardwareMonitor для доступа к сенсорам): будут нулевыми, планшет их
просто скроет. Загрузка CPU/GPU/RAM читается без прав.

**Файрвол.** При первом запуске Windows сама предложит «Разрешить доступ»
для NofsAgent.exe — это обычный диалог для программы (не требует прав
администратора, достаточно нажать «Разрешить» для частных сетей). Если
диалог не появился (или заблокирован политикой) и планшет пишет «нет
связи», хотя автопоиск находит ПК, — добавить правило вручную (нужны
права админа только для этого шага):

```powershell
netsh advfirewall firewall add rule name="NOFS Desk" dir=in action=allow protocol=TCP localport=48484
```

## config.json (рядом с exe)

- `port` — порт WebSocket (48484), `discoveryPort` — UDP-автопоиск (48485).
- `repoPath` — путь к git-репозиторию для панели Git.
- `github.repo` — `"owner/name"`; пусто — берётся из `git remote origin`.
- `github.token` — Personal Access Token (scope `repo`); пусто — только публичные репо.
- `apps` — список наблюдения для чипов контекста (по умолчанию Word, PowerPoint,
  Excel, Rider, Android Studio, Chrome): чип показывается **только когда приложение
  запущено**, активное окно подсвечено; тап — фокус на окно. `process` — имя
  процесса без .exe, `path` — чем запускать, если тапнули по чипу закрытого
  приложения (пусто — не запускать).
- `remoteType.hotkey` — клавиша-тоггл режима «клавиатура ПК → планшет»:
  `scrolllock` (по умолчанию) / `pause` / `f13` / `f14` / `f15`.
- `macros` — кнопки макросов. Поле `app`: пусто — системный макрос (секция
  «Система» на планшете), иначе id приложения из `apps` — макрос показывается,
  только когда это приложение в фокусе на ПК (секция «Макросы · Имя»).
  По умолчанию: хоткеи Word/PowerPoint/Excel (сохранить, печать, показ…),
  Rider/Android Studio (собрать, запуск, отладка, Gradle Sync, Logcat)
  и Chrome (вкладки, инкогнито, DevTools). `action`:
  - `run:программа|аргументы` — запуск (`run:wt.exe`, `run:ms-settings:nightlight`,
    `run:C:\dev\proj\gradlew.bat|assembleDebug`);
  - `keys:комбинация` — клавиши (`keys:win+printscreen`, `keys:ctrl+shift+esc`) —
    уходят в активное окно, поэтому app-макросы шлются только приложению в фокусе;
  - `lock` / `sleep` / `mute` — блокировка / сон / глушение звука.

После правки конфига перезапустить агента (меню трея → Выход).

## Трей

Серая иконка — планшет не подключён, зелёная (sage) — подключён.
Меню: статус, открыть config.json, открыть лог (`logs/agent.log`), выход.

## «Отправить на планшет» из Проводника

При старте агент регистрирует пункт контекстного меню для папок (HKCU, без
прав админа): правый клик по папке → «Отправить на планшет (NOFS)» — эта папка
становится git-репозиторием панели: планшет сразу получает ветки, историю
и GitHub-данные, выбор запоминается в config.json.

## Звук и плейтайм

Правый слот планшета переключается: Git / Звук / Время.
- **Звук** — микшер через Core Audio: общая громкость, мьют, кнопка мьюта
  микрофона, per-app ползунки по активным аудио-сессиям.
- **Время** — плейтайм-трекер: агент копит секунды активного окна по процессам
  из `apps` (в т.ч. игры) в `playtime.json` рядом с exe; планшет показывает
  «сегодня» и «за 7 дней». Данные старше 60 дней подчищаются.

Игровой сетап: добавь игру в `apps` (процесс без .exe) и макросы с
`"app": "<id>"` — при активной игре планшет покажет её кнопки.
Осторожно с онлайн-играми со строгим античитом: синтетические клавиши
(SendInput) там могут расцениваться как читерство.

## Сборка с планшета («Тень билда»)

Сборка запускается **макросом** с действием `build:<id>`, где `<id>` — запись из
раздела `builds` в config.json. Такой макрос рисуется зелёной кнопкой-пилюлей в
гриде макросов (иконка по типу: молоток=build, стрелка=run, жук=debug) и по тапу
открывает полноэкранную сцену. По умолчанию у Rider и Studio есть кнопка «Собрать»
(`build:agent` / `build:android`); остальные макросы добавляешь сам.

У каждой записи `builds[]`: `cmd` (программа), `args`, `ide`, `cwd`.
**Путь проекта:** если задан `ide` (`"studio"` | `"rider"`) — берётся из ТЕКУЩЕГО
открытого проекта этой IDE (агент читает её `recentProjects.xml` и берёт проект
с самым свежим timestamp). Так сборка следует за тем, что открыто в Rider/Studio,
без правки конфига. Если `ide` пуст или проект не найден — запасной вариант `cwd`.
В папке проекта должны лежать `gradlew.bat` (gradle) или `.sln`/`.csproj` (dotnet).

cmd-обёртки автоматически получают `chcp 65001`, чтобы вывод шёл в UTF-8
(иначе служебные сообщения cmd в OEM-кодировке превращаются в кракозябры).

Сборки из IDE (Rider/Studio) агент ловит по всплеску CPU gradle-демона/MSBuild
и показывает сцену «сборка в IDE» без деталей (опрос раз в секунду).

## Клавиатура ПК → планшет (remote typing)

Хоткей из `remoteType.hotkey` (по умолчанию **Scroll Lock** — почти нигде не
занят) переключает режим: пока он включён, физическая клавиатура ПК не
доходит до активного окна ПК вообще — низкоуровневый хук (`WH_KEYBOARD_LL`
в `RemoteTypeService`) глотает все нажатия и шлёт их на планшет, в поле
сообщения коммита (Git-панель → Commit). Раскладка (в т.ч. кириллица)
берётся из окна, которое было в фокусе на момент включения. **Escape** —
запасной выключатель, если забыли про хоткей. Режим включается только
хоткеем вручную (без автоматики) — так исключена утечка нажатий, и
автоматически выключается, если планшет отключился.

## Паспорт файла — анализаторы

`passportAnalyzer` в config.json: `auto` (по умолчанию) | `roslyn` | `treesitter` | `grep`.
- `auto` — C# через Roslyn (точный AST), Kotlin/Java/Scala через tree-sitter
  (если есть нативная грамматика), остальное — grep.
- `roslyn` — только Roslyn для C#, иначе grep. `treesitter` — tree-sitter для
  C#/Kotlin/Java/Scala, иначе grep. `grep` — везде эвристики.
- Ставь `roslyn` и `treesitter` на одном C#-файле, чтобы сравнить.

C# и Java/Scala-грамматики уже в пакете `TreeSitter.DotNet`. **Kotlin в пакет
не входит** — нужна нативная `tree-sitter-kotlin.dll` (win-x64) рядом с exe.
Собрать (нужны git, Node.js, C-компилятор — например из Build Tools for VS):

```powershell
npm install -g tree-sitter-cli
git clone https://github.com/fwcd/tree-sitter-kotlin
cd tree-sitter-kotlin
tree-sitter build --output tree-sitter-kotlin.dll
```

Положи получившуюся `tree-sitter-kotlin.dll` в папку с `NofsAgent.exe`
(или рядом с проектом при `dotnet run`). Без неё Kotlin молча падает на grep
(в логе — `tree-sitter .kt: ... (нужен tree-sitter-kotlin.dll рядом с exe)`).

## Git-панель

Планшет показывает полную историю (до 200 коммитов всех веток) с графом,
метками веток, переключением ветки (тап по имени ветки → список → checkout),
плюс Pull / Commit / Push. Медиа-плеер скрывается, когда на ПК ничего не играет.

## Автозапуск

Планировщик задач Windows → «Создать задачу» → триггер «При входе в систему»,
галка «Выполнять с наивысшими правами», действие — путь к NofsAgent.exe.

## Протокол

Один WebSocket `ws://пк:48484/ws`, JSON-сообщения с полем `type`
(hello/metrics/media/context/macros/git/github/remoteTypeState/remoteKey ←
агент; `{"type":"cmd", ...}` → агент).
C#-модели в `Protocol.cs` — зеркало Kotlin в `app/.../net/Protocol.kt`, менять синхронно.
Метрики и медиа пушатся раз в секунду (обложка — только при смене трека),
контекст — раз в 2 с, git/github — по запросу планшета.
