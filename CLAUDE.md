# NOFS Desk — контекст проекта для агента

## Что это
Персональная companion-панель: Android-планшет (Kotlin/Jetpack Compose, альбомный
canvas 1280×800, Teclast P40HD) + Windows PC-агент (C#, .NET 8, трей), связь по
WebSocket через WiFi. Философия проекта: «зеркало событий» — ПК-зависимый режим,
где события пользователя на ПК отражаются на планшете; планшет помогает и даёт
одно-тапные обратные действия, но не мешает. Не «кнопки на стекле», а живой пульт
с честной обратной связью: контрол отражает реальное состояние ПК (кнопка
«Жирный» горит, когда курсор в жирном тексте, и гаснет сама).

## Текущая ветка: feature/core (рез до ядра)
После фазы «наделали фич — не складывается в продукт» (ветка feature/carve)
всё отрезано до ядра. Резалось по принципу «убрать из склейки»: код многих фич
лежит в репо (components/, Services/), но не грузится и не рендерится —
возвращать точечно и осознанно.

## Архитектура (устоявшиеся решения)
- Клиент: Jetpack Compose, пастельный flat UI, сворачиваемая шапка, чёрный
  slide-in плеер справа, обложка морфится из круга в скруглённый квадрат (M3).
- Seam для данных: интерфейс `DeskDataSource`; `FakeDeskDataSource` (демо) ↔
  `WebSocketDeskDataSource` меняются настройкой.
- Связь: один WebSocket на все виджеты, разные message types и частоты.
  Метрики/медиа — push 1 с; контекст окна — 0.5 с; состояния макросов — 0.25 с
  (пуш только при смене сигнатуры).
- Рефлективные макросы: агент считает «активность» каждого макроса по факту ПК
  (GetKeyState для caps/num/scroll, Core Audio mute, GSMTC «играет», Word через
  COM late binding — GetActiveObject + dynamic, без Interop-зависимостей).
  Планшет подсвечивает кнопку, даёт хаптику и пружинку нажатия.
- Агент: SMTC/GSMTC (медиа), LibreHardwareMonitor (метрики), Core Audio
  (NAudio — сейчас только состояния mute), SendInput/Process.Start (макросы).
- Реконнект железобетонный: heartbeat (ping 15 с) + автопереподключение с
  бэкоффом (сон планшета, смена IP, WiFi-роуминг) — известная зона смерти.
- Протокол: `agent/Protocol.cs` ↔ `android/.../net/Protocol.kt` — зеркала,
  менять строго синхронно.

## ЯДРО — реализовано и работает (не трогать, фундамент)
1. Медиаплеер — SMTC, управление, чёрная пилюля внизу по центру → slide-in
   плеер; фолбэк на локальную медиа-сессию планшета, когда ПК молчит.
2. Системные метрики — CPU/GPU/RAM, полная сетка или спарклайны у часов.
3. Контекстные рефлективные макросы — per-app кнопки (Word/PPT/Excel/Rider/
   Studio/Chrome/игры) + системные; рефлективные состояния: caps, num, mute,
   media, word.bold/italic/underline (флагман обратной связи).
4. Скринсейвер — чёрный экран с часами по бездействию (таймаут в настройках).
5. Телефонная раскладка — урезанный экран (часы+статус, макросы, плеер
   оверлеем), портрет и альбом.
6. Настройки подключения — демо-режим, UDP-автопоиск агента, ручной IP;
   ошибки агента снекбаром.

## Удалено насовсем (не возвращать, не предлагать)
- Remote typing (клавиатура ПК → планшет) — решение пользователя: не нужно.
- Вырезанная волна идей: «тень билда», QR-мост буфера, «паспорт файла»,
  полноэкранный live-статус — не реализовывать и не предлагать (код паспорта
  и сцены физически ещё лежит, см. выше, но развивать нельзя).
- Плагинная архитектура; presence-«занят для окружающих»; LogLens — отдельный
  проект, сюда не тащить.

## В РАБОТЕ
- Лимиты Claude на планшете: агент дёргает `npx ccusage blocks --json`
  (раз в минуту), показывает токены 5-часового окна + время сброса + неделю;
  калибровка процентов — пользователь вводит «в приложении Anthropic сейчас N%»,
  агент вычисляет потолок (limit = tokens / N * 100) и хранит в config.

## БЭКЛОГ — подтверждённые идеи (не реализованы)

### Волна 1 (dev-инструменты)
1. Obsidian-интеграция — агент читает vault с диска; чекбоксы `- [ ]` как to-do
   на планшете с обратной синхронизацией (тап → построчная правка файла без
   потери форматирования); заметки автотегаются веткой/проектом через daily note.
2. Предиктивные алерты по метрикам — «через ~8 мин своп при таком росте памяти»;
   корреляция событий (диск+CPU = indexing в Rider).
3. Git-граф: «что изменилось пока меня не было», быстрый review диффа —
   надстройка над отключённой git-панелью, если вернём.
4. Second-screen для длинных логов — build output / логи на планшет.
5. Presence-режим — тихая синхронизация с календарём, без popup.

### Волна «зеркало событий» (ПК-зависимый режим)
6. Обмен файлами свайпом — файл/картинка с ПК на планшет; свайп на планшете
   кладёт картинку в буфер обмена ПК (сторона свайпа — в конфиге).
7. Вахтёр загрузок — плашка при загрузке файла (FileSystemWatcher на Downloads,
   прогресс по росту `.crdownload`/`.part`); по завершении — «открыть»/«показать».
8. Мониторинг процессов на портах — кто на каком порту слушает
   (`GetExtendedTcpTable`/`netstat -ano`); тап → открыть Swagger на ПК.
9. События USB-устройств — подключилось/отключилось + список.
10. Отлов диалогов ошибок — WinEvent-хук на `#32770` со словами
    Error/Exception/«Ошибка» → скрин окна на планшет.
11. Зеркало прогресс-баров копирования Explorer — тем же механизмом, что п.10:
    один механизм, две фичи.
12. AFK-режим — при тишине клавиатуры/мыши ПК планшет живёт сам: часы, погода,
    статистика дня, случайный TODO/FIXME. Частично есть (скринсейвер планшета);
    нет триггера от активности ПК и PresenceState-машины.

## Архитектурные заметки для реализации
- Единая шина событий: агент вешает хуки (SetWinEventHook, FileSystemWatcher,
  WMI process events, clipboard listener) → типизированные события в
  существующий WebSocket. Не плодить каналы под каждую фичу.
- `PresenceState` как единая state-машина: работаю → AFK → залочен
  (`SessionSwitch`). Три состояния, три экрана, один источник. Закладывать сразу.
- Панель активного окна — базовый примитив всех «зеркальных» фич. Сейчас
  поллинг 500 мс в ContextService; целевое — `EVENT_SYSTEM_FOREGROUND`-хук.
- Обратная связь макросов запуска: CLI — exit code; GUI — существование
  процесса/окна. (Не реализовано; сейчас есть рефлективные состояния-тумблеры.)
